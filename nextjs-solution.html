Next.js Build Error Solution

Resolving "Could not find a production build in the '.next' directory"

Error Message

Error: Could not find a production build in the '.next' directory. Try building your app with 'next build' before starting the production server.

This error occurs when you try to start a Next.js production server without first building the application.

Solution

You need to build your Next.js application before starting the production server:

npm
yarn
npm run build
npm start
yarn build
yarn start

Note: If you're in a development environment, you might want to use the development server instead, which automatically handles building:

npm run dev

Understanding the Problem

Next.js has two main modes:

  • Development Mode: Uses next dev with hot reloading for development
  • Production Mode: Uses next start to serve a pre-built application

The error occurs because you're trying to start the production server without first creating a production build.

Package.json Configuration

Ensure your package.json includes the following scripts:

{
  "scripts": {
    "dev": "next dev",
    "build": "next build",
    "start": "next start"
  }
}

Additional Tips

  • Always run npm run build before deploying your Next.js application
  • Check that your build completes without errors
  • If you're using a custom server (like Express), ensure it's properly configured
  • Verify that your .next directory isn't listed in .gitignore if you're deploying manually
